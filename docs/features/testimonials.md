# Feature: Testimonials

## Context

Social proof is disproportionately persuasive for non-technical buyers. A business owner evaluating a solo dev has no way to assess code quality — they rely on trust signals. Even one or two genuine quotes from colleagues, managers, or past collaborators can significantly increase conversion.

## Scope

**In scope:**
- Testimonials section on the homepage (high visibility) and/or services page
- Design for 3–6 cards initially, built to scale
- Each card: quote, person's name, role/company, optional photo

**Out of scope:**
- Third-party review integrations (Google, Clutch)

## Open Questions

- Where to source the first testimonials — colleagues at NAV IT, school project collaborators, anyone from past freelance work?
- Display as a static grid or horizontal scroll/carousel?
- Homepage section or services page only?
