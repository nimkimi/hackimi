# Feature: Pricing Page

## Context

Transparent pricing (even rough ranges) is one of the most effective ways to pre-qualify leads and signal confidence. Clients who contact you after seeing pricing are far more likely to convert. Hiding pricing forces a discovery call just to answer "can I afford this?" — which wastes both sides' time.

## Scope

**In scope:**
- New `/pricing` route or section within services page
- Present 2–3 engagement tiers or project type ranges:
  - Landing page / brochure site (e.g. €1–3k)
  - Web application / custom site (e.g. €3–10k)
  - Ongoing retainer / maintenance (e.g. €X/mo)
- Note what's included at each level (pages, rounds of revisions, support period)
- Clear caveat: "Prices vary based on scope — book a call for a custom quote"
- CTA → inquiry form or book a call

**Out of scope:**
- Real-time quotes or calculators (separate feature)
- Fixed prices — ranges only

## Open Questions

- Standalone `/pricing` page or embedded in services page?
- Use EUR or NOK (or both)? NOK is more natural for local clients; EUR for international.
- Display as a comparison table, card tiers, or plain prose?
