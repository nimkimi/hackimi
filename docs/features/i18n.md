# Feature: i18n (English + Norwegian)

## Context

The portfolio targets small business clients in Oslo. Norwegian-speaking business owners are significantly more likely to engage with a site in their own language. Adding Norwegian also demonstrates i18n capability — a marketable skill for clients who need multilingual sites.

## Scope

**In scope:**
- Library: `next-intl` — standard for Next.js 15 App Router (Server Component support, no client bundle overhead for static pages)
- Routing: URL-based locale prefix (`/en/`, `/no/`) — clean, SEO-friendly, easy to link between languages
- All user-facing strings extracted into `messages/en.json` and `messages/no.json`
- Language switcher component in the header (EN | NO toggle)
- Pages to translate: homepage, about, projects, contact — and any new pages as they're built
- `src/lib/site.ts` constants that appear in UI (role, description) need locale-aware variants
- `middleware.ts` for locale detection and redirect

**Out of scope:**
- Right-to-left languages
- Machine translation (strings should be written manually or reviewed by a native speaker)

## Technical Notes

- Next.js App Router i18n with next-intl uses a `[locale]` dynamic segment: `src/app/[locale]/`
- `next.config.js` needs the `createNextIntlPlugin` wrapper
- `buildPageMetadata()` in `src/lib/metadata.ts` needs locale-aware title/description
- OG images and JSON-LD structured data should reflect the active locale
